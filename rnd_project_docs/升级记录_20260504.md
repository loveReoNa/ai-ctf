## 二、各模块详述

### 1. MCP Integration（MCP 集成层）

**目标**：提供与外部安全工具的标准化 MCP 协议接口，统一工具发现、调用和结果处理。

**关键组件**：

| 组件 | 文件 | 说明 |
|------|------|------|
| MCPClient | `mcp_client.py` | 管理与 MCP 服务器的 stdio/HTTP 连接 |
| MCPConnectionConfig | `mcp_client.py` | 连接配置数据类（server_name, transport_type, command, args） |
| MCPServerRegistry | `mcp_server_registry.py` | 注册/发现/管理多 MCP 服务器 |
| ToolSchemaParser | `tool_schema_parser.py` | 将原始工具定义转换为标准 ToolSchema，支持质量过滤 |
| ToolQualityFilter | `tool_schema_parser.py` | 评估工具质量（description 完整性、参数数量等） |
| CTFAgentOrchestrator | `agent_orchestrator.py` | Agent 主循环编排，管理攻击阶段转换与工具调用 |
| RunConfig | `agent_orchestrator.py` | 运行配置（max_turns, target_host, model_name 等） |
| RunSummary | `agent_orchestrator.py` | 运行结果摘要 |
| AgentState | `schemas/agent_state_schema.py` | Agent 状态机，跟踪当前攻击阶段 |
| AttackPhase | `schemas/agent_state_schema.py` | 枚举：RECON→SCANNING→EXPLOITATION→POST_EXPLOITATION→REPORTING |
| ErrorRecord | `schemas/agent_state_schema.py` | 错误记录与恢复信息 |
| VulnInfo | `schemas/agent_state_schema.py` | 漏洞发现记录 |
| LoggedToolCall | `schemas/tool_call_schema.py` | 已记录的工具调用（tool_name, arguments, result, success, token_usage） |
| TokenUsage | `schemas/tool_call_schema.py` | Token 消耗追踪（input_tokens, output_tokens） |
| PromptManager | `prompts/prompt_manager.py` | 加载/管理 5 类 CTF 攻击模板提示词 |

### 2. Benchmark Framework（基准测试框架）

**目标**：提供可重复、可比较的 CTF 能力测试框架，支持 LiveMCPBench 和 XBow 两种评测协议。

**关键组件**：

| 组件 | 文件 | 说明 |
|------|------|------|
| ToolWrapper | `tool_wrapper.py` | 统一包装安全工具调用，集成输出过滤 |
| LiveMCPBenchParser | `parsers.py` | 解析 LiveMCPBench JSON/YAML 配置与结果 |
| XBowLogParser | `parsers.py` | 解析 XBow 原生输出日志 |
| FlagMaskingSanitizer | `output_sanitizer.py` | 基于正则的 Flag 值掩码，防止评测泄漏 |
| ToolClassifier | `tool_category.py` | 基于关键词匹配的 7 类工具自动分类 |
| ToolCategory | `tool_category.py` | 枚举：侦察(1)→漏洞扫描(2)→漏洞利用(3)→密码破解(4)→后渗透(5)→信息收集(6)→其他(7) |

### 3. Visualization Dashboard（可视化仪表盘）

**目标**：提供 Docker 隔离沙箱、挑战自动部署、XBow 集成运行和批量测试编排能力。

**关键组件**：

| 组件 | 文件 | 说明 |
|------|------|------|
| SandboxManager | `sandbox_manager.py` | Docker 沙箱全生命周期管理（创建/监控/销毁） |
| SandboxConfig | `sandbox_manager.py` | 沙箱配置（image, cpu_limit, memory_limit, network_mode, timeout_seconds） |
| SandboxInfo | `sandbox_manager.py` | 沙箱运行时信息（container_id, ip_address, ports） |
| SandboxStatus | `sandbox_manager.py` | 枚举：CREATING→RUNNING→STOPPED→ERROR→DESTROYED |
| TargetDeployer | `target_deployer.py` | 将 CTF 挑战部署到 Docker 环境中 |
| ChallengeDefinition | `target_deployer.py` | 挑战定义（name, category, difficulty, flag_pattern, time_limit_seconds） |
| XBowRunner | `xbow_runner.py` | 驱动 XBow 进行自动化 CTF 测试 |
| XBowConfig | `xbow_runner.py` | XBow 运行配置（bin_path, model_name, timeout_seconds, retry_count） |
| LogCollector | `log_collector.py` | 多源日志聚合（Agent 输出 + XBow 输出 + 沙箱日志） |
| BatchRunner | `batch_runner.py` | 批量执行多个挑战，支持并行和失败重试 |

### 4. Data Analysis（数据分析增强）

**目标**：在现有基础上增强数据解析、指标计算、基线对比、可视化图表和报告导出能力。

**关键新增/增强**：

| 组件 | 文件 | 说明 |
|------|------|------|
| LogParser（增强） | `log_parser.py` | 支持 `parse_agent_log()` 结构化解析，标记 flag/success/tool_calls/tokens |
| MetricsEngine（新增） | `metrics_engine.py` | **9 指标评估体系**：准确性(accuracy, precision, recall, f1) + 效率(task_completion_rate, avg_solve_time) + 智能性(first_attempt_success_rate, hallucination_rate) + 覆盖度(challenge_type_coverage, vuln_types_mastered) |
| BaselineManager（新增） | `baseline.py` | **5 条基线对比**：human_expert, expert_writeup, random_solver, gpt4_baseline, tool_only_baseline |
| Visualization（增强） | `visualization.py` | **15 种图表** + `create_sample_charts()` 便捷函数 |
| ReportGenerator（新增） | `report_generator.py` | **3 种格式导出**：JSON（结构化数据）、Markdown（可读文本）、HTML（交互式报告） |
