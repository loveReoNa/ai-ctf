# AI-CTF 推理决策清晰与可解释性优化方案

> **任务分配**：本方案由 **组员 A（伍祥）** 与 **组员 B** 共同负责实施  
> **关联文件**：《毕设项目改进升级计划.md》（模块一 agent_orchestrator、模块三 log_collector、模块四 data_analysis）  
> **核心目标**：在用户输入解题指令后，项目运行的全过程中，清晰、结构化地展示 Agent 的推理链与决策依据，使每一轮"思考→选工具→执行→评估"的过程对人完全透明可解释

---

## 一、问题分析 —— 当前项目的推理可解释性痛点

### 1.1 现有推理机制回顾

当前项目（`src/cai/`）已具备基础的推理能力：

| 机制 | 文件 | 能力 | 不足 |
|------|------|------|------|
| `thought()` 工具 | `src/cai/tools/misc/reasoning.py` | Agent 可记录 breakdowns / reflection / action / next_step / key_clues | 输出为纯文本拼接，无结构化，无时间轴，仅 Agent 自身调用 |
| `think()` 工具 | 同上 | 通用的"思考"日志追加 | 过于简单，仅返回传入的文本 |
| ThoughtAgent | `src/cai/agents/thought.py` | 独立的分析/规划 Agent，可调用其他 Agent 并交替思考 | 仅在 multi-agent 模式下使用，多数场景下未启用 |
| 系统提示词 | `src/cai/prompts/system_thought_router.md` | 定义了 Thought() → AnotherAgent() → Thought() 循环 | 提示词粒度不够精细，无强制结构化输出要求 |
| 增强推理显示 | `src/cai/util.py` | 对 Claude/DeepSeek 等模型的 thinking/reasoning 内容提供 Rich Panel 展示 | 仅对支持 reasoning 的模型生效，不是所有场景 |

### 1.2 核心问题

```
用户输入 "CTF_NAME=hackableii cai" → 终端开始输出工具调用结果
                                    ↓
              终端显示大量工具返回报文（nmap 扫描结果、curl 响应等）
                                    ↓
                   用户难以区分 "Agent 为什么这样做" vs "工具返回了什么"
                                    ↓
              整个解题过程缺乏结构化的推理可视化，像"黑盒在执行"
```

具体表现为：

1. **推理与执行混杂**：Agent 的思考过程（WHY）与工具调用结果（WHAT）混在同一输出流中，缺乏清晰分层
2. **决策依据不可追溯**：Agent 为什么在某一轮选择工具 A 而非工具 B？为什么放弃某条攻击路径？这些决策点没有显式记录
3. **攻击阶段不可感知**：用户无法一目了然地知道当前处于侦察/利用/提权/Flag捕获的哪个阶段
4. **进度不可量化**：用户在等待过程中无法判断"Agent 进展了百分之多少""还有多久"
5. **事后无法复盘**：虽然有计划中的 JSON 日志（见改进升级计划 7.2 节），但缺乏"解题过程回放"的能力

## 二、优化方案总体设计

### 2.1 设计原则

1. **分层展示**：推理层（WHY）、决策层（WHICH）、执行层（WHAT）、评估层（RESULT）四层清晰分离
2. **实时可见**：解题过程中实时输出推理链，而非事后查看日志
3. **结构化可解析**：所有推理输出遵循统一 Schema，既可用于人阅读，也可被程序解析生成报告
4. **非侵入性**：不改变 Agent 的核心执行逻辑，在现有架构上增加"推理可观测层"
5. **渐进可扩展**：从简单到复杂分阶段实现，确保每个阶段都有可见产出

### 2.2 总体架构（起于"推理可观测层"）

```
┌─────────────────────────────────────────────────────────────────────┐
│                      用户终端 (TUI / CLI)                              │

│  │ 推理面板 │  │ 攻击阶段指示器 │  │ 决策树   │  │ 工具执行结果面板  │  │
│  │ (WHY)   │  │ (PHASE)       │  │ (CHOICE) │  │ (WHAT)           │  │
```

#### 3.2.1 `reasoning_schema.py` — 推理链数据结构（核心）

该文件定义所有推理相关实体的标准化数据结构，是整个"推理可解释"方案的基础。

```python
"""
推理链数据结构定义
定义 Agent 解题过程中所有推理/决策相关的标准化 Schema
"""
from dataclasses import dataclass, field
from datetime import datetime
from enum import Enum
from typing import Optional, List, Dict, Any


class ReasoningEntryType(Enum):
    """推理条目类型"""
    SITUATION_ASSESSMENT = "situation_assessment"     # 态势评估
    HYPOTHESIS_GENERATION = "hypothesis_generation"    # 假设生成
    TOOL_SELECTION = "tool_selection"                  # 工具选择决策
    RESULT_INTERPRETATION = "result_interpretation"     # 结果解释
    STRATEGY_SHIFT = "strategy_shift"                  # 策略转变
    FLAG_REASONING = "flag_reasoning"                  # Flag 相关推理
    ERROR_ANALYSIS = "error_analysis"                  # 错误分析


@dataclass
class ReasoningStep:
    """
    单步推理记录 —— 这是最核心的数据结构
    
    对应系统提示词中的 THOUGHT 输出块
    """
    step_id: str                            # 推理步骤唯一标识
    timestamp: datetime                     # 时间戳
    turn_number: int                        # 所属 Agent 轮次
    entry_type: ReasoningEntryType          # 推理类型
    agent_name: str                         # 执行推理的 Agent
    
    # 核心推理内容
    situation: str                          # 当前态势描述：Agent 看到了什么
    hypothesis: str                         # 核心假设：Agent 认为可能是什么
    confidence: float                       # 置信度 0.0 ~ 1.0
    
    # 决策内容
    selected_action: str                    # 选择的行动方案
    alternative_actions: List[str] = field(default_factory=list)  # 考虑过的替代方案
    selection_rationale: str = ""           # 选择理由：为什么选这个而非其他
    
    # 预期与结果
    expected_outcome: str = ""              # 预期的结果
    actual_outcome: str = ""                # 实际结果（执行后回填）
    outcome_match: Optional[bool] = None    # 结果是否符合预期
    
    # 关键发现
    key_findings: List[str] = field(default_factory=list)  # 本轮关键发现
    flags_found: List[str] = field(default_factory=list)   # 发现的 Flag
    
    # 元信息
    token_usage: Optional[Dict[str, int]] = None   # 此步 Token 消耗

        self.steps.append(step)

        # 根据 entry_type 和 situation 推断阶段
        ...
    
    def to_display_tree(self) -> dict:
        """将推理链转换为可视化树结构"""
        ...


@dataclass
class AttackPhaseTransition:
    """攻击阶段转换记录"""
```

#### 3.2.2 `observation_hooks.py` — 事件钩子系统

该文件在 `CTFAgentOrchestrator`（改进升级计划模块一）的关键节点植入钩子，拦截事件并转发给格式化引擎。

```python
"""
事件钩子系统 —— 在 Agent 编排器各节点植入观测点

对应改进升级计划 3.2.4 agent_orchestrator.py 中的 CTFAgentOrchestrator
在 execute_turn、select_tools、execute_tool_calls、evaluate_progress
这些方法的关键位置触发钩子
"""
from typing import Callable, List, Optional
from cai.observability.reasoning_schema import ReasoningStep, ReasoningEntryType


class ObservationHooks:
    """
    推理可观测性钩子管理器
    
    使用方式（在 CTFAgentOrchestrator 中）：
        hooks = ObservationHooks(formatter=ReasoningFormatter())
        orchestrator = CTFAgentOrchestrator(config, registry, hooks=hooks)
        
    钩子触发点：
        1. on_turn_start        —— 每轮开始
        2. on_situation_analyze —— Agent 分析当前态势后
        3. on_hypothesis_made   —— Agent 形成假设后
        4. on_tool_selected     —— Agent 选定工具后
        5. on_tool_executed     —— 工具执行完毕后
        6. on_result_evaluated  —— Agent 评估结果后
        7. on_strategy_shifted  —— Agent 改变策略时
        8. on_phase_changed     —— 攻击阶段变化时
        9. on_flag_found        —— 发现 Flag 时
        10. on_error            —— 发生错误时
    """
    
    def __init__(self):
        self._hooks: dict[str, List[Callable]] = {
            "turn_start": [],
            "situation_analyze": [],
            "hypothesis_made": [],
            "tool_selected": [],
            "tool_executed": [],
            "result_evaluated": [],
            "strategy_shifted": [],
            "phase_changed": [],
            "flag_found": [],
            "error": [],
        }
        self._current_step: Optional[ReasoningStep] = None
        self._reasoning_chain = None  # ReasoningChain 引用
    
    def register(self, event: str, callback: Callable) -> None:
        """注册事件回调"""
        if event in self._hooks:
            self._hooks[event].append(callback)
    
    def _trigger(self, event: str, **kwargs) -> None:
        """触发事件"""
        for callback in self._hooks.get(event, []):
            try:
                callback(**kwargs)
            except Exception as e:
                # 钩子异常不应影响解题主流程
                print(f"[Hook Error] {event}: {e}")
    
    # ---- 各触发器方法 ----
    
    def on_turn_start(self, turn_number: int, phase: str, available_tools: List[str]):
        """Agent 进入新的一轮"""
        self._current_step = ReasoningStep(
            step_id=f"step_{turn_number:04d}",
            timestamp=datetime.now(),
            turn_number=turn_number,
            entry_type=ReasoningEntryType.SITUATION_ASSESSMENT,
            agent_name="",
            situation=f"进入第 {turn_number} 轮，当前阶段: {phase}",
            hypothesis="",
            confidence=0.0,
            selected_action="",
        )
        self._trigger("turn_start", turn_number=turn_number, phase=phase, 
                       available_tools=available_tools)
    
    def on_situation_analyze(self, situation: str, key_observations: List[str]):
        """Agent 完成态势分析"""
        if self._current_step:
            self._current_step.situation = situation
            self._current_step.key_findings = key_observations
        self._trigger("situation_analyze", situation=situation, 
                       observations=key_observations)
    
    def on_hypothesis_made(self, hypothesis: str, confidence: float, 
                            alternatives: List[str]):
        """Agent 形成攻击假设"""
        if self._current_step:
            self._current_step.hypothesis = hypothesis
            self._current_step.confidence = confidence
            self._current_step.entry_type = ReasoningEntryType.HYPOTHESIS_GENERATION
        self._trigger("hypothesis_made", hypothesis=hypothesis, 
                       confidence=confidence, alternatives=alternatives)
    
    def on_tool_selected(self, tool_name: str, rationale: str, 
                          alternatives_considered: List[str]):
        """Agent 选定了要调用的工具"""
        if self._current_step:
            self._current_step.selected_action = tool_name
            self._current_step.selection_rationale = rationale
            self._current_step.alternative_actions = alternatives_considered
            self._current_step.entry_type = ReasoningEntryType.TOOL_SELECTION
        self._trigger("tool_selected", tool_name=tool_name, 
                       rationale=rationale, alternatives=alternatives_considered)
    
    def on_tool_executed(self, tool_name: str, success: bool, 
                          result_summary: str, execution_time_ms: float):
        """工具执行完毕"""
        if self._current_step:
            self._current_step.actual_outcome = result_summary
            self._current_step.execution_time_ms = execution_time_ms
        self._trigger("tool_executed", tool_name=tool_name, success=success,
                       result_summary=result_summary, execution_time_ms=execution_time_ms)
    
    def on_result_evaluated(self, expected_matched: bool, 
                             interpretation: str, next_intent: str):
        """Agent 评估了工具执行结果"""
        if self._current_step:
            self._current_step.outcome_match = expected_matched
        self._trigger("result_evaluated", expected_matched=expected_matched,
                       interpretation=interpretation, next_intent=next_intent)
    
    def on_strategy_shifted(self, old_strategy: str, new_strategy: str, 
                             reason: str):
        """Agent 改变了攻击策略"""
        self._trigger("strategy_shifted", old_strategy=old_strategy,
                       new_strategy=new_strategy, reason=reason)
    
    def on_phase_changed(self, from_phase: str, to_phase: str, 
                          trigger: str):
        """攻击阶段发生变化"""
        self._trigger("phase_changed", from_phase=from_phase,
                       to_phase=to_phase, trigger=trigger)
    
    def on_flag_found(self, flag: str, location: str):
        """发现 Flag"""
        self._trigger("flag_found", flag=flag, location=location)
    
    def on_error(self, error_type: str, error_message: str, 
                  recovery_action: str):
        """发生错误"""
        current = self._current_step
        if current:
            current.entry_type = ReasoningEntryType.ERROR_ANALYSIS
        self._trigger("error", error_type=error_type, 
                       error_message=error_message, recovery_action=recovery_action)
```

#### 3.2.3 `reasoning_formatter.py` — 推理格式化引擎

该文件负责将推理链数据结构转化为用户终端上的可视化输出。

```python
"""
推理格式化引擎
将 ReasoningStep 数据结构转化为 Rich 终端的可视化输出"""

from typing import List
from datetime import datetime
from cai.observability.reasoning_schema import ReasoningStep, AttackPhaseTransition

# 依赖: rich (已在 pyproject.toml 中)
try:
    from rich import print as rprint
    from rich.panel import Panel
    from rich.table import Table
    from rich.text import Text
    from rich.console import Console
    from rich.layout import Layout
    from rich.live import Live
    from rich.progress import Progress, SpinnerColumn, TextColumn
    from rich import box
    HAS_RICH = True
except ImportError:
    HAS_RICH = False


class ReasoningFormatter:
    """
    推理格式化引擎
    
    负责将结构化的推理数据转换为以下展示形式：
    1. 实时终端面板（解题过程中）
    2. 推理链摘要文本（日志持久化）
    3. Markdown 推理报告（事后复盘）
    """
    
    def __init__(self, verbose: bool = True, color: bool = True):
        self.verbose = verbose
        self.color = color
        self.console = Console() if HAS_RICH else None
    
    # ===== 实时终端展示 =====
    
    def format_reasoning_panel(self, step: ReasoningStep) -> Panel:
        """
        将单步推理格式化为 Rich Panel
        
        输出样例：
        ┌─── 🤔 Agent 推理 · 第 5 轮 ───────────────────────────┐
        │                                                        │
        │  📍 当前态势                                           │
        │  目标 192.168.100.10 上发现开放端口:                    │
        │  - 22/tcp  (SSH - OpenSSH 7.4)                        │
        │  - 80/tcp  (HTTP - Apache 2.4.6)                      │
        │  - 8080/tcp (HTTP - Tomcat 9.0)                       │
        │                                                        │
        │  💡 核心假设 (置信度: 85%)                              │
        │  Tomcat 9.0 可能存在 CVE-2019-0232 CGIServlet RCE       │
        │  → 优先尝试该漏洞的已知利用方式                          │
        │                                                        │
        │  🎯 选择的行动                                         │
        │  工具: metasploit_tomcat_cgi                           │
        │  理由: 与版本精确匹配，且无需认证，成功率评估高          │
        │  替代方案曾考虑:                                         │
        │    · gobuster 路径爆破（可能耗时较长）                  │
        │    · SQL 注入探测（端口未见数据库服务）                   │
        │                                                        │
        │  ⏱ 预期结果: 获得初始 shell 访问                        │
        └────────────────────────────────────────────────────────┘
        """
        if not HAS_RICH:
            return self._format_plain_text(step)
        
        # 构建面板内容
        content = Text()
        
        # 态势
        content.append("📍 当前态势\n", style="bold cyan")
        content.append(f"{step.situation}\n\n", style="white")
        
        # 假设
        conf_color = "green" if step.confidence >= 0.7 else "yellow" if step.confidence >= 0.4 else "red"
        content.append(f"💡 核心假设 ", style="bold yellow")
        content.append(f"(置信度: {step.confidence*100:.0f}%)\n", style=conf_color)
        content.append(f"{step.hypothesis}\n\n", style="white")
        
        # 决策
        content.append("🎯 选择的行动\n", style="bold green")
        content.append(f"工具: {step.selected_action}\n", style="green")
        content.append(f"理由: {step.selection_rationale}\n", style="white")
        if step.alternative_actions:
            content.append("替代方案曾考虑:\n", style="dim")
            for alt in step.alternative_actions:
                content.append(f"  · {alt}\n", style="dim")
        
        # 预期
        if step.expected_outcome:
            content.append(f"\n⏱ 预期结果: {step.expected_outcome}\n", style="italic cyan")
        
        title = f"🤔 Agent 推理 · 第 {step.turn_number} 轮"
        return Panel(content, title=title, border_style="blue", box=box.ROUNDED)
    
    def format_result_panel(self, step: ReasoningStep) -> Panel:
        """
        将工具执行结果与推理验证格式化为面板，与推理面板形成对照
        
        输出样例：
        ┌─── 📊 执行结果 · 第 5 轮 ─────────────────────────────┐
        │                                                        │
        │  🔧 工具: metasploit_tomcat_cgi                        │
        │  ⏱ 耗时: 3.42s                                        │
        │  ✅ 执行成功                                            │
        │                                                        │
        │  📋 结果摘要:                                          │
        │  成功利用 CVE-2019-0232，获得 www-data 用户 shell       │
```

#### 3.3.1 `system_structured_reasoning.md` — 结构化推理系统提示词

这是实现推理可解释性的**最关键文件**。它强制 Agent 在每一轮都输出结构化的推理块。

```markdown
# 结构化 CTF 解题推理系统提示词

你是一个具备 **透明推理链** 的 CTF 解题 Agent。
你的每一次思考和决策都必须以结构化、可追溯的方式呈现。

---

## 核心原则

1. **推理与执行分离**：先完整表达思考过程，再执行工具调用
2. **假设驱动**：每一轮基于显式假设行动，而非盲目尝试
3. **置信度标注**：对每个假设给出 0%~100% 的置信度
4. **备选方案显式化**：列出考虑过但未选择的方案并说明为何不选
5. **结果闭环验证**：每一轮结束后，对比实际结果与预期，更新置信度

---

## 每轮强制输出格式

你必须严格按照以下 6 个区块输出，不得省略：

### 区块 1: SITUATION（态势评估）

```

当前可用的关键信息: <凭据/hash/文件路径等>
```

### 区块 2: HYPOTHESIS（核心假设）

```

假设: <我认为存在什么漏洞/攻击路径>
置信度: <0-100%>

```

### 区块 6: NEXT（下一步意图）

```
[NEXT]
下一步: <接下来要做什么>

#### 3.3.2 `decision_tracer.py` — 决策树追踪与回放

该文件记录 Agent 的完整决策路径，支持解题结束后进行决策树可视化和回放分析。

```python
"""
决策树追踪器 —— 记录 Agent 完整决策路径，支持回放分析

核心功能：
1. 构建决策树：每个节点 = 一个决策点（工具选择、策略改变）
2. 标注成功/失败分支
3. 支持事后回放：按时间轴重放整个解题过程的推理
4. 生成决策摘要图：展示 Agent"走了哪些路"
"""
from dataclasses import dataclass, field
from typing import List, Optional, Dict
from datetime import datetime
from enum import Enum


class BranchOutcome(Enum):
    SUCCESS = "success"           # 此分支导致进展
    FAILURE = "failure"           # 此分支失败
    ABANDONED = "abandoned"       # 此分支被放弃（Agent 主动转向）
    UNEXPLORED = "unexplored"     # 此分支未有实质尝试


@dataclass
class DecisionNode:
    """决策树节点"""
    node_id: str                            # 唯一标识
    turn_number: int                        # 轮次
    parent_node_id: Optional[str]           # 父节点
    
    # 决策内容
    hypothesis: str                         # 当时持有的假设
    confidence: float                       # 当时置信度
    selected_action: str                    # 选择的行动
    alternatives: List[str] = field(default_factory=list)  # 未选的分支
    
    # 执行结果
    outcome: BranchOutcome = BranchOutcome.UNEXPLORED
    result_summary: str = ""
    
    # 时间
    timestamp: datetime = field(default_factory=datetime.now)
    
    # 子节点
    children: List['DecisionNode'] = field(default_factory=list)


@dataclass
class DecisionTree:
    """完整决策树"""
    run_id: str
    root: Optional[DecisionNode] = None
    all_nodes: Dict[str, DecisionNode] = field(default_factory=dict)
    current_node: Optional[DecisionNode] = None
    
    def add_decision(self, turn: int, hypothesis: str, confidence: float,
                     selected: str, alternatives: List[str]) -> DecisionNode:
        """添加一个新的决策节点"""
        node = DecisionNode(
            node_id=f"node_{turn:04d}",
            turn_number=turn,
            parent_node_id=self.current_node.node_id if self.current_node else None,
            hypothesis=hypothesis,
            confidence=confidence,
            selected_action=selected,
            alternatives=alternatives,
        )
        
        if self.current_node:
            self.current_node.children.append(node)
        if self.root is None:
            self.root = node
        
        self.all_nodes[node.node_id] = node
        self.current_node = node
        return node
    
    def update_outcome(self, node_id: str, outcome: BranchOutcome, 
                        summary: str):
        """更新节点的执行结果"""
        if node_id in self.all_nodes:
            node = self.all_nodes[node_id]
            node.outcome = outcome
            node.result_summary = summary
    
    def add_abandoned_branch(self, parent_node_id: str, 
                               description: str, reason: str):
        """记录一个被显式放弃的分支（Agent 考虑过但决定不尝试）"""
        # 在父节点下添加一个 ABANDONED 子节点记录
        node = DecisionNode(
            node_id=f"abandoned_{len(self.all_nodes)}",
            turn_number=-1,
            parent_node_id=parent_node_id,
            hypothesis=description,
            confidence=0.0,
            selected_action=description,
            outcome=BranchOutcome.ABANDONED,
            result_summary=f"放弃理由: {reason}",
        )
        if parent_node_id in self.all_nodes:
            self.all_nodes[parent_node_id].children.append(node)
        self.all_nodes[node.node_id] = node
    
    def to_ascii_tree(self) -> str:
        """生成 ASCII 决策树可视化
        
        输出样例:
        🎯 决策树: CTF 解题全路径
        ├── [第1轮] 初始侦察 (✅)
        │   ├── 放弃: SSH弱口令尝试 → 端口未确认
        │   └── 放弃: masscan → 缺少版本信息
        ├── [第2轮] 服务版本探测 (✅)
        │   └── 发现 Apache 2.4.6 + Tomcat 9.0.0.M1
        ├── [第3轮] 尝试 Tomcat 默认凭据 (❌)
        │   └── 转向: 查找 Tomcat 版本相关 CVE
        ├── [第4轮] 弃用方案: SSH 暴力破解 (放弃)
        │   └── 理由: Web 类题目通常入口在 Web 层
        └── [第5轮] CVE-2019-0232 利用 (✅)
            └── 获得 www-data shell → 进入后渗透阶段
        """
        ...
    
    def to_replay_sequence(self) -> list:
        """生成按时间排序的回放序列，每条包含完整的推理上下文"""
        ...


class DecisionTreeBuilder:
    """
    决策树构建器 —— 与 ObservationHooks 配合使用
    
    用法：
        hooks = ObservationHooks()
        tree_builder = DecisionTreeBuilder(run_id="...")
        hooks.register("tool_selected", tree_builder.on_tool_selected)
        hooks.register("tool_executed", tree_builder.on_tool_executed)
        hooks.register("strategy_shifted", tree_builder.on_strategy_shifted)
    """
    
    def __init__(self, run_id: str):
        self.tree = DecisionTree(run_id=run_id)
    
    def on_tool_selected(self, tool_name: str, rationale: str, 
                          alternatives: List[str], **kwargs):
        """当 Agent 选定工具时，记录决策节点"""
        node = self.tree.add_decision(
            turn=kwargs.get("turn_number", 0),
            hypothesis=kwargs.get("hypothesis", ""),
            confidence=kwargs.get("confidence", 0.0),
            selected=tool_name,
            alternatives=alternatives,
        )
        # 如果 alternatives 中有被放弃的，也记录
        if alternatives and rationale:
            for alt in alternatives:
                self.tree.add_abandoned_branch(
                    node.node_id, alt, f"选择 {tool_name} 优先于 {alt}"
                )
    
    def on_tool_executed(self, tool_name: str, success: bool, 
                          result_summary: str, **kwargs):
        """当工具执行完毕时，更新结果"""
        if self.tree.current_node:
            outcome = BranchOutcome.SUCCESS if success else BranchOutcome.FAILURE
            self.tree.update_outcome(
                self.tree.current_node.node_id, outcome, result_summary
            )
    
    def on_strategy_shifted(self, old_strategy: str, new_strategy: str, 
                              reason: str, **kwargs):
        """当 Agent 改变策略时，记录为分支切换"""
        if self.tree.current_node:
            self.tree.update_outcome(
                self.tree.current_node.node_id, 
                BranchOutcome.ABANDONED,
                f"策略转换: {old_strategy} → {new_strategy}。原因: {reason}"
            )
```

#### 3.3.3 `phase_tracker.py` — 攻击阶段追踪器

```python
"""
攻击阶段追踪器 —— 实时追踪并展示 Agent 所处攻击阶段

攻击链阶段:

from dataclasses import dataclass, field
from datetime import datetime

class AttackPhase(Enum):
    RECONNAISSANCE = "recon"           # 端口扫描、主机发现
    ENUMERATION = "enumeration"        # 服务版本探测、目录爆破、漏洞扫描
    EXPLOITATION = "exploitation"      # 漏洞利用、获取初始访问
    POST_EXPLOITATION = "post_exploit" # 权限提升、横向移动、信息收集
    FLAG_CAPTURE = "flag_capture"      # 搜索并捕获 Flag


@dataclass
class PhaseTransition:
    """阶段转换事件"""
    from_phase: Optional[AttackPhase]
    to_phase: AttackPhase
    trigger: str                        # 触发转换的事件描述
    timestamp: datetime
    turn_number: int
    confidence: float = 1.0             # 转换的置信度


@dataclass 
class PhaseTrackerState:
    """阶段追踪器状态"""
    current_phase: AttackPhase = AttackPhase.RECONNAISSANCE
    transitions: List[PhaseTransition] = field(default_factory=list)
    phases_visited: List[AttackPhase] = field(default_factory=list)
    flags_found: int = 0
    target_identified: bool = False
    initial_access_obtained: bool = False
    root_obtained: bool = False
    
    @property
    def progress_percentage(self) -> float:
        """估算解题进度百分比（基于阶段和关键里程碑）"""
        progress_map = {
            AttackPhase.RECONNAISSANCE: 0.1,
            AttackPhase.ENUMERATION: 0.3,
            AttackPhase.EXPLOITATION: 0.5,
            AttackPhase.POST_EXPLOITATION: 0.75,
            AttackPhase.FLAG_CAPTURE: 0.9,
        }
        base = progress_map.get(self.current_phase, 0.0)
        if self.flags_found > 0:
            base = min(0.95, base + 0.05 * self.flags_found)
        return base


class AttackPhaseTracker:
    """
    攻击阶段追踪器
    
    通过分析 Agent 的行为模式，自动推断当前所处的攻击阶段。
    规则：
    - 使用 nmap/masscan 等扫描工具 → 侦察阶段
    - 使用 gobuster/nikto/sqlmap 等检测工具 → 枚举阶段
    - 使用 metasploit/exploit 类工具 → 利用阶段
    - 使用 linpeas/提权工具 → 后渗透阶段
    - 使用 find/cat 搜索 flag → Flag捕获阶段
    """
    
    PHASE_TOOL_MAP = {
        AttackPhase.RECONNAISSANCE: ["nmap", "masscan", "ping", "arp-scan", "netdiscover"],
        AttackPhase.ENUMERATION: ["gobuster", "dirb", "nikto", "wfuzz", "ffuf", 
                                   "whatweb", "sqlmap", "enum4linux"],
        AttackPhase.EXPLOITATION: ["msfconsole", "searchsploit", "metasploit",
                                    "python3 -c", "exploit", "reverse_shell"],
        AttackPhase.POST_EXPLOITATION: ["linpeas", "linenum", "linux-exploit-suggester",
                                         "sudo -l", "find / -perm", "getcap"],
        AttackPhase.FLAG_CAPTURE: ["find / -name", "cat /root", "grep -r flag",
                                    "cat /home", "ls /root"],
    }
    
    def __init__(self):
        self.state = PhaseTrackerState()
    
    def analyze_tool_for_phase(self, tool_name: str, arguments: str = "") -> Optional[AttackPhase]:
        """
        根据工具名称和参数推断属于哪个攻击阶段
        
        返回 None 表示无法推断（如用 curl 可能属于多个阶段）
        """
        combined = f"{tool_name} {arguments}".lower()
        
        for phase, keywords in self.PHASE_TOOL_MAP.items():
            for keyword in keywords:
                if keyword in combined:
                    return phase
        return None
    
    def update(self, tool_name: str, arguments: str = "", 
               result_text: str = "", turn_number: int = 0) -> bool:
        """
        根据工具调用更新阶段
        
        返回 True 表示阶段发生了变化
        """
        inferred_phase = self.analyze_tool_for_phase(tool_name, arguments)
        
        if inferred_phase is None:
            return False
        
        if inferred_phase != self.state.current_phase:
            old_phase = self.state.current_phase
            transition = PhaseTransition(
                from_phase=old_phase,
                to_phase=inferred_phase,
                trigger=f"工具 {tool_name} 的调用表明进入 {inferred_phase.value} 阶段",
                timestamp=datetime.now(),
                turn_number=turn_number,
            )
            self.state.transitions.append(transition)
            self.state.current_phase = inferred_phase
            self.state.phases_visited.append(inferred_phase)
            return True
        return False
    
    def mark_milestone(self, milestone: str):
        """标记关键里程碑"""
        if milestone == "initial_access":
            self.state.initial_access_obtained = True
        elif milestone == "root":
            self.state.root_obtained = True
        elif milestone.startswith("flag:"):
            self.state.flags_found += 1
    
    def get_current_phase_display(self) -> dict:
        """获取当前阶段的终端展示数据"""
        phase_order = [
            AttackPhase.RECONNAISSANCE,
            AttackPhase.ENUMERATION,
            AttackPhase.EXPLOITATION,
            AttackPhase.POST_EXPLOITATION,
            AttackPhase.FLAG_CAPTURE,
        ]
        phase_names = {
            AttackPhase.RECONNAISSANCE: "🔍 侦察",
            AttackPhase.ENUMERATION: "📋 枚举",
            AttackPhase.EXPLOITATION: "💥 利用",
            AttackPhase.POST_EXPLOITATION: "⬆️ 后渗透",
            AttackPhase.FLAG_CAPTURE: "🚩 Flag捕获",
        }
        
        phases_status = []
        current_idx = phase_order.index(self.state.current_phase)
        
        for i, phase in enumerate(phase_order):
            if i < current_idx:
                status = "completed"
            elif i == current_idx:
                status = "active"
            else:
                status = "pending"
            phases_status.append({
                "name": phase_names[phase],
                "status": status,
                "index": i,
            })
        
        return {
            "phases": phases_status,
            "progress": self.state.progress_percentage,
            "flags_found": self.state.flags_found,
            "root_obtained": self.state.root_obtained,
            "current_phase_name": phase_names[self.state.current_phase],
        }
```
